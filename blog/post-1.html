用 Three.js 做一个可旋转的 3D 虚拟服装展示

把一件衣服放进网页,让用户用鼠标自由旋转、缩放查看,是数字时尚最直观的展示方式之一。本文用 Three.js 搭一个最小可运行示例,不依赖任何打包工具,直接用 ES Module 即可跑起来;最后再聊怎么把它做成产品级的虚拟试穿入口。

1. 为什么要在网页里展示 3D 服装

相比一张渲染图,可交互的 3D 模型能传达褶皱走向、版型体积和面料垂感——这些是平面图很难表达的。而对研究者来说,网页还意味着零安装、跨平台、易分享:把链接发给合作者,对方点开就能转。它也是连接"建模"和"展示/评测"的自然桥梁。

2. 准备模型

先用 Blender 或 CLO 3D 导出 .glb(GLTF 的二进制版,体积小、无需额外贴图文件)。放到项目的 models/ 目录前,注意几点:

  • 单位与轴向:统一用米,且让"上"为 +Y;很多服装软件导出时轴向不一致,会在网页里"躺平"。
  • 面数预算:网页端建议单件衣服控制在 2–5 万面以内;过高会拖慢中低端手机。
  • 原点:把模型原点放在衣服几何中心附近,后续相机"框住"物体会简单很多。

3. 场景、相机、渲染器(最小骨架)

Three.js 的三件套:场景(Scene)、相机(PerspectiveCamera)、渲染器(WebGLRenderer)。把它们挂到一个容器 div 上,并让相机看向原点。

import * as THREE from 'three'; import { OrbitControls } from 'three/addons/controls/OrbitControls.js'; const stage = document.getElementById('stage'); const scene = new THREE.Scene(); const camera = new THREE.PerspectiveCamera(45, 1, 0.1, 100); camera.position.set(0, 1, 3); const renderer = new THREE.WebGLRenderer({ antialias: true, alpha: true }); renderer.setSize(480, 480); renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2)); stage.appendChild(renderer.domElement);

4. 加载模型 + 轨道控制器

GLTFLoader 读入模型,OrbitControls 负责"拖拽旋转、滚轮缩放"。加载完成后把模型加入场景,并用一个辅助函数让相机自动框住物体。

const controls = new OrbitControls(camera, renderer.domElement); controls.enableDamping = true; new GLTFLoader().load('models/garment.glb', (g) => { scene.add(g.scene); fitCameraToObject(g.scene, camera, controls); }); function animate() { requestAnimationFrame(animate); controls.update(); renderer.render(scene, camera); } animate();

其中 fitCameraToObject 用包围盒(BoundingBox)算出一个合适距离,避免"衣服太大撑出屏幕"或"太小看不见"。网上有成熟写法,直接复用即可。

5. 打光与环境

服装材质对光很敏感。用 HemisphereLight 做环境光,加一盏 DirectionalLight 勾出褶皱阴影;若用 MeshStandardMaterial,建议再叠加基于图像的照明(IBL,如 RoomEnvironment),布料质感会真实很多。

scene.add(new THREE.HemisphereLight(0xffffff, 0x888888, 1.1)); const key = new THREE.DirectionalLight(0xffffff, 1.6); key.position.set(2, 4, 3); scene.add(key); import { RoomEnvironment } from 'three/addons/environments/RoomEnvironment.js'; const pmrem = new THREE.PMREMGenerator(renderer); scene.environment = pmrem.fromScene(new RoomEnvironment(), 0.04).texture;

6. 让展示更"产品级"

到这里已经能转能看了。再往前走几步,就能做成可用的展示位:

  • 自动旋转:controls.autoRotate = true,无人操作时缓慢自转,有人交互时停下。
  • 换色:遍历材质改 .color,或预设几套配色让用户点选。
  • 布料/版型切换:预载多个 glb,切换时淡入淡出,做成"虚拟试穿"雏形。
  • 移动端手势:OrbitControls 默认支持单指旋转、双指缩放,基本开箱即用。

7. 体积与性能

模型大了加载慢。几个立竿见影的优化:

  • 导出时开启 Draco 几何压缩 与 KTX2 纹理压缩,体积常能降 70%+。
  • 用 GLTFLoader 的 setDRACOLoader / setKTX2Loader 配套解码。
  • 多件衣服共用同一材质时,用实例化减少 draw call。

小结

至此,一个可旋转、可缩放、带真实打光的 3D 服装展示就完成了。它的输入是一个 .glb 文件,而"如何把一堆衣服批量、标准化地导出来喂给这里"正是下一篇要讲的内容——见 《用 Blender 批量导出服装数字资产》。
