可穿戴系统入门:从柔性传感器到数据可视化

可穿戴系统的本质,是把人体信号变成可读的数据。听起来高深,但拆开看,它几乎总是一条相同的链路:感知 → 调理/采集 → 传输 → 可视化。本文用这条链路串起选型思路与常见坑,给第一次搭系统的你一个地图。

1. 一条最小可行链路

不论做的是心率衣、姿态裤还是手势手套,都可以画成同一张图:

柔性传感器 → 信号调理 → MCU(ADC) → 无线(BLE) → 手机/网页 → 图表

每一环都有现成模块,真正难的不是"让灯亮",而是让数据稳定、可解释。下面逐环说。

2. 柔性传感器选型

常见的柔性传感机制有三种,各有侧重:

  • 电阻/压阻式(如导电织物、Velostat):受压电阻变化,便宜、易缝进布料,但迟滞和非线性明显。
  • 电容式:对接近/形变敏感,功耗低、动态好,但易受湿度与电磁干扰。
  • 应变片/液态金属:量程与精度更好,工艺门槛高一些。

经验法则:先做原型验证选压阻织物,追求精度再上电容或应变方案。

3. 信号调理与采集

传感器出来的信号往往很弱、很脏。调理电路做三件事:

  • 放大:用仪表放大器(INA)把 mV 级信号抬到 ADC 友好区间。
  • 滤波:硬件 RC 低通先挡掉高频噪声,软件再做平滑。
  • 偏置:把双极性信号搬到 ADC 的 0–Vref 范围内。

MCU 侧用 ADC 采样,注意采样率与分辨率够用即可——动作信号几十 Hz 就够,不必盲目追高。

4. 无线传输

wearable 多半用 BLE(Bluetooth Low Energy):手机直连、功耗低。把采样值打包成特征值(Characteristic)周期广播,网页端可用 Web Bluetooth 直接收(注意仅 HTTPS/localhost 可用)。

// 伪代码:每 50ms 发一帧 setInterval(() => { const frame = readSensors(); // 读 ADC characteristic.writeValue(frame); // BLE 发出 }, 50);

5. 数据可视化

数据到手机/网页后,先画出来才有意义。最小方案:

  • 串口绘图:开发阶段用串口监视器看波形,最快定位问题。
  • 网页图表:用轻量库(如 Chart.js)做滚动折线,实时显示各通道。
  • 标注事件:让用户点"开始动作"打标,后面做动作识别才有标签。

6. 常见坑

  • 运动伪影:衣服滑动、电缆拽动都会伪造信号。固定走线、贴紧皮肤能缓解。
  • 基线漂移:温度/出汗让零点慢慢漂。要做归零/高通。
  • 供电噪声:电机、开关电源会串进模拟端。模拟/数字分开供电、加磁珠。
  • 蓝牙丢包:别把帧发太大太密,留出余量。

小结

把系统想成一条链路,每环都可独立替换与标定,问题就好定位了。其中传感器标定与噪声处理往往最耗时,下一篇就以柔性压力传感器为例,拆一遍完整流程——见 《柔性压力传感器的标定与噪声处理》。
