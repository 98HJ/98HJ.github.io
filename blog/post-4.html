2026.04 · 服装数字化 · Digital Fashion

用 Blender 批量导出服装数字资产

做数字服装常要在 Blender 里管理几十件 garment,手工导出又慢又易错。本文用一段 Python 脚本把"导出"变成一键操作:遍历集合、统一规范、批量写出标准 glTF,直接喂给网页或游戏引擎。

1. 为什么需要批量

当资产数量上十,手工点"导出"会出三种问题:命名不一致、原点/单位忘了统一、漏导或重复导。脚本把这些变成确定性过程——同样输入永远同样输出,可重复、可审查。

2. 先定资产规范

批量前先约定,后面才不乱:

  • 格式:glTF / GLB(自包含、引擎友好)。
  • 单位:米;轴向:+Y 向上。
  • 原点:每件的几何中心,方便下游"框住"。
  • 命名:garment_<类别>_<编号>.glb,如 garment_jacket_01.glb。

3. 遍历集合导出

把每一件衣服放在独立 Collection,脚本遍历它们逐个导出:

import bpy, os out = "/path/to/assets" for col in bpy.data.collections: # 只处理以 garment_ 开头的集合 if not col.name.startswith("garment_"): continue # 隐藏其他,仅显示当前集合 for c in bpy.data.collections: c.hide_render = (c is not col) bpy.ops.export_scene.gltf( filepath=os.path.join(out, col.name + ".glb"), use_selection=False, export_format='GLB', export_yup=True, export_draco_mesh_compression_enable=True, ) print("exported", col.name)

4. 导出前自动整理

在循环里顺手做几件"脏活",省得回头改:

  • 重命名:按规范统一 object 名,避免中文/空格。
  • 应用变换:bpy.ops.object.transform_apply 把位移/旋转/缩放烘进网格,下游不会再"歪"。
  • 减面:对超模可用 decimate 修改器压到目标面数。

5. 校验与清单

导出完别急着关。生成一份清单,至少核对:

  • 每个 glb 都能被 GLTFLoader 加载(写个最小化网页批量试)。
  • 文件大小在预期区间(异常大往往是忘了压纹理)。
  • 命名与数量与源集合一一对应。

6. 接入下游

标准 glb 几乎到处能进:网页用 Three.js(见前文)、Unity/Unreal 直接导入、Blender 也能再读回。因为规范统一,切换展示端几乎零改造成本。

小结

批量导出不是"偷懒",而是把资产管线标准化。规范定一次,脚本写一次,之后每次加新衣服都是"丢进集合、跑脚本"。当衣服多到几十上百件,这套流程会省下大量返工。
