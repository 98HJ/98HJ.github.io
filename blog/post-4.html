2026.04 · 服装数字化 · Fashion Digitization

用 Blender 批量导出服装数字资产

做数字服装常要在 Blender 里管理几十件 garment,手工导出又慢又易错。用脚本批量转成标准 .glb,能直接喂给下游引擎(见上一篇的 Three.js 展示)。

1. 组织场景

把每件衣服放在独立的 Collection(如 Garment_A / Garment_B),命名即资产名,方便脚本按名遍历。

2. 批量导出脚本

import bpy, os out = "/path/to/assets" for col in bpy.data.collections: if not col.name.startswith("Garment"): continue bpy.ops.export_scene.gltf( filepath=os.path.join(out, col.name + ".glb"), export_format='GLB', use_active_collection=True, export_selected=False )

3. 统一规范

· 单位设为米,避免比例错乱;
· 烘焙法线 / 合并材质,减小体积;
· 保留顶点色或 UV,方便下游换色。

4. 下游对接

导出的 .glb 可直接进 Three.js、Unity 或虚幻。若要做虚拟试穿,记得保留骨骼与蒙皮信息,否则衣服无法随人体动作变形。

小结

把"导出"变成一行命令,你就能把精力放在设计本身,而不是重复的点击上。资产管线一旦标准化,迭代速度会明显提升。
